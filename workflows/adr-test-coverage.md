# ADR Test Coverage Workflow

Validates that every Architecture Decision Record (ADR) has corresponding test cases that would fail if the architectural commitment was violated. Flags gaps and creates stub TC-IDs.

---

## Step 0: Workspace Resolution

@~/.claude/sdlc/workflows/workspace-resolution.md

After resolution:
```bash
PHASE_ARTIFACTS="$ARTIFACTS/design"
mkdir -p "$PHASE_ARTIFACTS"
```

---

## Step 1: Read ADRs and Test Cases

Read in parallel:
- `$ARTIFACTS/design/solution-design.md` — primary ADR source
- `$ARTIFACTS/test-cases/test-cases.md` — existing test coverage

Skip condition: If solution-design.md does not exist → log `{ "status": "skipped-condition-not-met", "summary": "solution-design.md not found" }` and output `⏭️ adr-test-coverage — skipped: no ADRs found`

---

## Step 2: Extract ADR Commitments

For each ADR-NNN in solution-design.md:
  Extract the core commitment from the "Decision" section — one sentence capturing what the system commits to architecturally.

---

## Step 3: Match ADRs to Test Cases

Search test-cases.md for TC-IDs that:
  - Explicitly reference the ADR number
  - Test the behavior the ADR commits to

Produce a coverage table:
| ADR | Commitment | Covering TC-IDs | Status |
|-----|-----------|-----------------|--------|

---

## Step 4: Create Stubs for Uncovered ADRs

For each ADR with no test coverage: append to test-cases.md (or create `$ARTIFACTS/test-cases/adr-coverage-stubs.md` if test-cases.md doesn't exist):

```
TC-ADR-NNN: Validate ADR-NNN — [decision title]
Layer: [appropriate layer]
Given: The system is configured per ADR-NNN ([one-line decision])
When: [scenario that exercises the architectural commitment]
Then: [outcome that proves the ADR holds]
Note: Auto-generated by adr-test-coverage. Fill in specifics.
```

---

## Step 5: Write Report

Write `$PHASE_ARTIFACTS/adr-test-coverage.md`:

```markdown
# ADR Test Coverage Report
*Generated: [date] | Branch: [branch]*

## Summary
- ADRs evaluated: N
- Fully covered: N
- No test coverage: N (stubs created)

## Coverage Table
[table from Step 3]

## Stubs Created
[list TC-ADR-NNN entries created]
```

---

## Step 6: Update State

Append to autoChainLog:
```json
{
  "skill": "adr-test-coverage",
  "triggeredAfter": "adr-gen",
  "status": "completed",
  "artifact": "$PHASE_ARTIFACTS/adr-test-coverage.md",
  "summary": "N ADRs evaluated, M covered, K stubs created",
  "completedAt": "<ISO-timestamp>"
}
```

---

## Step 7: Output

If --auto-chain:
```
✅ adr-test-coverage — N ADRs checked, K coverage gaps → stubs created [path]
```

If interactive:
```
✅ ADR Test Coverage Complete

ADRs evaluated: N
  Covered by tests: N
  No test coverage (stubs created): N

[If gaps: list ADR titles with gap and the stub TC-IDs created]

Report: $PHASE_ARTIFACTS/adr-test-coverage.md
```
